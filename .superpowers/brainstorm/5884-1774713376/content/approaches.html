Three Approaches to Your Social Media System

Pick the architecture that matches your priorities

A

n8n Cloud + Supabase + Custom React Dashboard ⭐ Recommended

Stack: n8n Cloud → OpenAI/Claude → Supabase → your React /admin route

How it works: n8n Cloud runs all AI agent workflows (research, copy, image gen, scheduling). Supabase stores content drafts with real-time updates. Your existing React site gets an /admin route that queries Supabase — no extra hosting needed.

Pros

  • Zero infrastructure to manage
  • Dashboard lives inside your existing site
  • Supabase real-time = instant dashboard updates
  • Scales easily as posting frequency grows

Cons

  • n8n Cloud: ~$24–50/mo + Supabase free tier
  • Data stored on n8n + Supabase servers
B

Self-hosted n8n (Railway) + Supabase + Custom React Dashboard

Stack: n8n on Railway → OpenAI/Claude → Supabase → your React /admin route

How it works: Same architecture as A but you run n8n yourself on Railway ($5/mo). More control, lower long-term cost, but you handle n8n updates and uptime.

Pros

  • Cheapest long-term (~$5–10/mo total)
  • Full data control
  • Same clean React dashboard

Cons

  • You maintain n8n (updates, crashes)
  • Slightly more setup work upfront
C

Make.com + Airtable + Lightweight Embed

Stack: Make.com scenarios → OpenAI → Airtable → embedded Airtable views in /admin

How it works: Make.com (formerly Integromat) handles automation flows. Airtable acts as content database and review interface. Embed Airtable views or build minimal React wrappers around Airtable API.

Pros

  • Airtable is excellent for content review UX
  • Very fast initial setup
  • No custom backend code

Cons

  • Operations-based pricing can spike with volume
  • Dashboard feels less native in your site
  • Less flexible for custom AI agent logic
